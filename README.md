# Vite Module Federation Vue Repro

Minimal standalone repro for debugging Vite module federation with Vue.

## Background

Vue ships two distinct builds ([docs](https://vuejs.org/guide/scaling-up/tooling.html#note-on-in-browser-template-compilation)):

- `vue/dist/vue.runtime.esm-bundler.js` — runtime-only. No template compiler included. Templates must be pre-compiled by a build tool (e.g. `@vitejs/plugin-vue`). This is the default exposed via `package.json` exports.
- `vue/dist/vue.esm-bundler.js` — full build. Includes the template compiler for in-browser template compilation.

When a host needs to share Vue with remotes that require in-browser template compilation, it must point the shared module at the full build via the `import` option.

## The bug

When `import` is set on a shared singleton in the federation config, the build fails with `[MISSING_EXPORT]` errors for every Vue export. The virtual loadShare module generated by `@module-federation/vite` ends up not re-exporting any of Vue's named exports.

This is not limited to the full build. The same failure also reproduces when `import` points at the default runtime build (`vue/dist/vue.runtime.esm-bundler.js`), which is what Vue's `package.json` `exports` field resolves to by default.

**To reproduce:** uncomment the marked line in `vite.config.ts` and run `npm run build`.

```ts
shared: {
    vue: {
        singleton: true,
        import: 'vue/dist/vue.esm-bundler.js', // <-- this causes the bug
        // import: 'vue/dist/vue.runtime.esm-bundler.js', // <-- same bug with the default export
    },
},
```

**Expected:** build/dev server succeeds.

**Actual (build):** build fails with 9 `[MISSING_EXPORT]` errors:

**Actual (dev server):** the dev server starts, but the app fails to load. The browser console shows:

```
Uncaught (in promise) SyntaxError: The requested module '/@id/__x00__virtual:mf:__mfe_internal__host__loadShare__vue__loadShare__.js' does not provide an export named 'createElementBlock' (at App.vue:2:78)
```

## Investigation

Both Vue entry files re-export most of their public API via a star export from `@vue/runtime-dom`:

```js
// vue/dist/vue.esm-bundler.js
export * from '@vue/runtime-dom';
// ...
export { compileToFunction as compile };

// vue/dist/vue.runtime.esm-bundler.js
export * from '@vue/runtime-dom';
// ...
export { compile };
```

The missing exports (`createApp`, `createElementVNode`, `openBlock`, `createElementBlock`, etc.) are all provided through that `export * from '@vue/runtime-dom'` line. The virtual loadShare module generated by `@module-federation/vite` does not surface star re-exports from the shared entry.

By contrast, the explicit `export { compile }` at the bottom of each file is a direct named export on the entry module itself, and that export does come through. So the failure is specific to star re-exports, not to every export on the Vue entry file.
